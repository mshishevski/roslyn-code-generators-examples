# 02 - Strongly Typed API Client

## What This Generator Does

This example reads a deterministic `api-contract.json` file from `AdditionalFiles` and generates:

- request DTOs
- response DTOs
- a typed client interface
- an `HttpClient`-based implementation

## Input Model

- Additional file: `api-contract.json`
- Contract fields:
  - `clientName`
  - `namespace`
  - `endpoints[]` with `name`, `method`, `route`, `request`, `response`

## Generated Output

- `TypedApiClient.g.cs` in the contract namespace
- Generated types include:
  - `I{ClientName}`
  - `{ClientName}`
  - request/response DTO classes

All generated files include:

```csharp
// <auto-generated />
```

## Diagnostics

The generator emits explicit diagnostics for invalid/missing contract values, including:

- missing `clientName`
- missing `namespace`
- missing `endpoint` information
- invalid HTTP method
- missing request/response payload definitions

## Why This Is Realistic

This pattern is useful for:

- contract-first internal platform APIs
- generated SDKs for backend-to-backend integration
- reducing handwritten client drift
- compile-time safety with deterministic inputs

## Run The Example

From this folder:

```powershell
dotnet build StronglyTypedApiClient.sln
dotnet test StronglyTypedApiClient.sln
dotnet run --project src/StronglyTypedApiClient.Demo/StronglyTypedApiClient.Demo.csproj
```

The demo uses a fake `HttpMessageHandler`, so there are no network calls.
