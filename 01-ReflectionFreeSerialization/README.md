# 01 - Reflection-Free Serialization

## What This Generator Does

This example uses an incremental Roslyn source generator to discover DTOs marked with `GenerateJsonSerializableAttribute` and emit a `JsonSerializerContext` with `JsonSerializable` entries for each annotated type.

## Input Model

- C# attribute: `GenerateJsonSerializableAttribute`
- Input type shape: class/record/struct marked with `[GenerateJsonSerializable]`

## Generated Output

- `GeneratedJsonContext.g.cs` in namespace `ReflectionFreeSerialization.Demo.Generated`
- A partial context:
  - `public partial class GeneratedJsonContext : JsonSerializerContext`
  - Includes one `[JsonSerializable(typeof(...))]` attribute per annotated type

All generated files include:

```csharp
// <auto-generated />
```

## Why This Is Realistic

Reflection-free serialization is valuable for:

- Native AOT and trimming safety
- Lower startup overhead
- More predictable serialization behavior at build time
- Avoiding runtime reflection paths in high-performance services

## Run The Example

From this folder:

```powershell
dotnet build ReflectionFreeSerialization.sln
dotnet test ReflectionFreeSerialization.sln
dotnet run --project src/ReflectionFreeSerialization.Demo/ReflectionFreeSerialization.Demo.csproj
```

The demo serializes and deserializes using generated metadata:

```csharp
var json = JsonSerializer.Serialize(request, GeneratedJsonContext.Default.CreateOrderRequest);
```

## Notes

- No network calls are made by the generator.
- If no types are annotated, no `GeneratedJsonContext` is emitted.
- Generated source can be inspected under `obj/Generated` because `EmitCompilerGeneratedFiles` is enabled in the demo project.
